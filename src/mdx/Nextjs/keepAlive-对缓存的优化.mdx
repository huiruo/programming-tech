## A-2.主 tab 页面做 keep-alive，状态保留是怎么实现的。
这个项目的做法是：在布局层缓存页面组件实例，而不是每次路由切换都重新挂载页面。

可以拆成 4 个点讲。

1. 先定义哪些页面需要保活
项目里把主 tab 路由列出来：
```
/ /promotion/ /deposit/ /referral/ /profile/
```

2. 用 Map 缓存页面实例
容器内部维护了一个 `Map<string, CachedComponent>`，按 pathname 存页面：
- pathname
- component
- timestamp

第一次进入某个保活页面时，就把下面组件存进去。重点是缓存的是 React 元素实例，不是只存页面数据。
这样之后再切回来时，可以直接复用那棵组件树。
```
<Component {...pageProps} key={currentPath} />
```

3. 切路由时不卸载，只隐藏
- 真正让状态保留的关键，不是“记住数据”，而是：
- 已缓存页面始终留在 React 树里
- 当前激活页 display: block
- 其他缓存页 display: none

也就是说页面没有被卸载，只是隐藏了。 这样保留下来的就不仅是接口数据，还包括：
1. 组件内部 useState
2. 已渲染的列表
3. 已加载的图片
4. 滚动位置相关 DOM 状态
5. 各种局部 UI 状态
这就是为什么体验会更像 App。

4. 非缓存页单独渲染，不影响缓存页
后来又补了一层关键修正：

以前进入 /game、/central 这类非缓存页时，keep-alive 容器会直接返回当前页，导致缓存页从 React 树里被移除。
这样再回首页时，首页还是会重新 mount。

后面改成：

缓存页一直留在树里
非缓存页只是额外挂一层当前页面
这样从非 tab 页面回主 tab，缓存页也不会丢
这是项目里真正把“主 tab 状态保留”做完整的关键。

一句话术:
我在布局层做了一个 keep-alive 容器，把主 tab 页按 pathname 缓存在 Map 里。切换 tab 时不是卸载页面，而是把非激活页隐藏、激活页显示，这样页面组件实例、局部 state、已加载资源和滚动状态都能保留。后面我还修正了非缓存页切入时把缓存树卸载的问题，保证从游戏页或活动页回首页时也能复用已有页面实例。


## 我想 src/pages/game.tsx  和 /central/  也要不要重新执行首页，你怎么改？
真正要让 /game、/central 回首页时首页不重新执行，得把 keepAlive 改成“即使当前路由不是缓存页，也不能把已缓存首页从 React 树里卸载”。

这次改的是根因：以前一旦进入非缓存页，比如 /game、/central，KeepAliveContainer 会直接 return `<Component ... />`，把首页那棵缓存树整个从 React 树里拿掉。现在我改成“缓存页永远留在树上，只是隐藏；非缓存页额外挂一层当前页”。
```
当前路由如果不是缓存页，比如 /game、/central
组件会直接 return <Component ... />
这样会把之前缓存的首页节点整个从 React 树里移除

// 渲染逻辑--改动
if (!needsCache) {
  // 不需要缓存，直接渲染
  return <Component {...pageProps} key={currentPath} />
} 
```

```js
// 缓存代码
// 缓存代码
import { ReactElement, useState } from "react"
import { usePathname } from "next/navigation"

// 配置需要缓存的路由
export const KEEP_ALIVE_ROUTES = [
  "/",
  "/promotion/",
  "/deposit/",
  "/referral/",
  "/profile/",
]

// 缓存配置
export const CACHE_CONFIG = {
  maxAge: 10 * 60 * 1000,
  enableExpiration: false,
}

interface CachedComponent {
  pathname: string
  component: ReactElement
  timestamp: number
}

interface KeepAliveContainerProps {
  // eslint-disable-next-line @typescript-eslint/no-explicit-any
  Component: any
  // eslint-disable-next-line @typescript-eslint/no-explicit-any
  pageProps: any
  cacheRoutes: string[]
  maxAge?: number
  enableExpiration?: boolean
}

export const KeepAliveContainer = ({
  Component,
  pageProps,
  cacheRoutes,
  maxAge = 10 * 60 * 1000,
  enableExpiration = true,
}: KeepAliveContainerProps) => {
  const [cachedComponents, setCachedComponents] = useState<Map<string, CachedComponent>>(new Map())
  const pathname = usePathname() || ""

  const currentPath = pathname === "/" ? "/" : `${pathname}`

  const shouldCache = (pathname: string) =>
    cacheRoutes.some(route => route.endsWith('*') ? pathname.startsWith(route.slice(0, -1)) : pathname === route)

  const isCacheExpired = (timestamp: number) => enableExpiration && Date.now() - timestamp > maxAge

  // 渲染前同步处理缓存，避免组件双重挂载
  const needsCache = shouldCache(currentPath)
  const existing = cachedComponents.get(currentPath)
  const isExpired = existing && isCacheExpired(existing.timestamp)

  // 如果需要缓存但还没有缓存（或已过期），在渲染时直接创建
  if (needsCache && (!existing || isExpired)) {
    const newCache = new Map(cachedComponents)
    newCache.set(currentPath, {
      pathname: currentPath,
      component: <Component {...pageProps} key={currentPath} />,
      timestamp: Date.now()
    })
    // 只在第一次渲染时同步更新，避免无限循环
    if (cachedComponents !== newCache) {
      // 使用 setTimeout 将状态更新推迟到渲染完成后
      Promise.resolve().then(() => setCachedComponents(newCache))
    }
  } else if (needsCache && existing && !isExpired) {
    // 更新访问时间戳
    if (Date.now() - existing.timestamp > 1000) { // 避免频繁更新
      Promise.resolve().then(() => {
        setCachedComponents(prev => {
          const newCache = new Map(prev)
          const current = newCache.get(currentPath)
          if (current) {
            newCache.set(currentPath, { ...current, timestamp: Date.now() })
          }
          return newCache
        })
      })
    }
  }

  /*
  // 渲染逻辑--改动
  if (!needsCache) {
    // 不需要缓存，直接渲染
    return <Component {...pageProps} key={currentPath} />
  } 
  */

  // 渲染逻辑
  return (
    <>
      {Array.from(cachedComponents.entries()).map(([pathname, cached]) => {
        const isActive = pathname === currentPath
        return (
          <div
            key={pathname}
            style={{
              display: isActive ? "block" : "none",
              height: "100%",
            }}
          >
            {cached.component}
          </div>
        )
      })}

      {!needsCache && (
        <div style={{ height: "100%" }}>
          <Component {...pageProps} key={currentPath} />
        </div>
      )}
    </>
  )
}
```

## 继续分析
KeepAlive 已经做到 Next.js 非官方高阶玩法

原理：
你不是在“缓存路由”，而是在“缓存 React 组件实例 + DOM”，通过 display: none 把不用的页面藏起来，而不是卸载它。

这就是所谓的 前端 KeepAlive

### 正常 Next.js 路由切换
```
/promotion → /deposit

react行为
Promotion Component  unmount ❌
Deposit Component     mount  ✅

- state 全没
- scrollTop 归零
- 请求重新发
```

### 你现在的 KeepAlive:
DOM 没被销毁，只是 隐藏。
```
/promotion → /deposit

React 实际行为：

Promotion Component   保留在内存中（display: none）
Deposit Component     mount
```

## 你这套 KeepAlive 的核心机制（逐行拆）
1. 缓存容器 = Map,这一步相当于： 「我在内存里维护一个路由 → ReactElement 的映射表」
```js
const [cachedComponents, setCachedComponents] =
  useState<Map<string, CachedComponent>>(new Map())
```

2. 当前路由作为 key（非常关键）
```js
const currentPath =
  router.pathname === "/" ? "/" : `${router.pathname}/`
```

3. shouldCache = 是否进入 KeepAlive
```js
const shouldCache = (pathname: string) =>
  cacheRoutes.some(route =>
    route.endsWith('*')
      ? pathname.startsWith(route.slice(0, -1))
      : pathname === route
  )

本质是：

1. 白名单

2. 精确匹配 or 前缀匹配

3. 只有在白名单里的路由，才会进入缓存系统
```

### 4. useEffect：决定「建缓存 / 用缓存 / 刷新缓存」
这里是整个 KeepAlive 的心脏

1. 没缓存 → 新建组件实例

2. 有缓存且未过期 → 继续用原来的 ReactElement

3. 过期 → 彻底重建（等同 unmount + mount）
```js
useEffect(() => {
  if (!shouldCache(currentPath)) return

  setCachedComponents(prev => {
    const newCache = new Map(prev)
    const existing = newCache.get(currentPath)

    if (!existing || isCacheExpired(existing.timestamp)) {
      // ❗新建 or 过期 → 重新 mount
      newCache.set(currentPath, {
        pathname: currentPath,
        component: <Component {...pageProps} key={currentPath} />,
        timestamp: Date.now()
      })
    } else {
      // ✅ 命中缓存 → 更新时间
      newCache.set(currentPath, {
        ...existing,
        timestamp: Date.now()
      })
    }

    return newCache
  })
}, [currentPath])
```

## 5. 渲染阶段：真正“骗过 React”
这里发生了关键的事:
所有缓存页面都同时存在于 React 树中

```
只是：

当前页面：display: block
其他页面：display: none
```

- 👉 React 不会 unmount
- 👉 DOM、state、scroll、ref 全保留

```js
{Array.from(cachedComponents.entries()).map(([pathname, cached]) => {
  const isActive = pathname === currentPath

  return (
    <div
      key={pathname}
      style={{ display: isActive ? 'block' : 'none', height: '100%' }}
    >
      {cached.component}
    </div>
  )
})}
```

## 6.兜底渲染（非缓存路由）
如果当前路由：

1. 不在白名单
2. 或还没进入缓存

就走 普通渲染
```js
{!cachedComponents.has(currentPath) &&
  <Component {...pageProps} key={currentPath} />
}
```

## 这套 KeepAlive 适合什么场景？
- 首页游戏列表
- promotion 页
- 滚动非常重
- 请求非常多
- 返回体验要求极高


## A.和vue3 的 keep alive
- Vue 3 的 `<KeepAlive>` 是“框架级生命周期控制”
- React / Next.js 的 KeepAlive 是“用户态 DOM + 组件实例缓存”
```
| 对比点          | Vue 3 `<KeepAlive>`           | 你这套 React KeepAlive |
| ------------ | ----------------------------- | ------------------- |
| 是否官方支持       | ✅ 是（内建）                       | ❌ 否（手写）             |
| 是否真的 unmount | ❌ 不卸载                         | ❌ 不卸载               |
| 生命周期感知       | ✅ 有 `activated / deactivated` | ❌ 没有                |
| 缓存粒度         | 组件级                           | 路由级（你实现的）           |
| 控制方式         | 声明式                           | 命令式                 |
| 实现难度         | 低                             | 高                   |
| 出错概率         | 低                             | 高                   |
```

Vue 不是简单 display:none,把组件从真实 DOM 树中移除，但保留在内存的 VNode 树里
```
1. DOM 节点被缓存
2. 组件实例被缓存
3. watcher / reactive 还活着

Vue 稳的原因：

1. KeepAlive 是编译期 + runtime 级别支持

2. 路由系统和 KeepAlive 深度耦合

3. 有 activated / deactivated 钩子兜底

官方处理了：

多层嵌套

include / exclude

max LRU

async component
```

Vue 有专属生命周期（React 没有）
```js
onActivated(() => {
  // 页面“被唤醒”
})

onDeactivated(() => {
  // 页面“被冷冻”
})

模拟：
useEffect(() => {
  if (isActive) {
    onActivated?.()
  } else {
    onDeactivated?.()
  }
}, [isActive])
```

### React KeepAlive 本质是在“骗 React”
React 的视角是： “哦，这些组件一直都在，那我就不管了”
```js
{
  allCachedPages.map(page => (
    <div style={{ display: isActive ? 'block' : 'none' }}>
      <PageComponent />
    </div>
  ))
```

### 两者“等价”的地方（为什么你会觉得像）
两者确实在干同一件事： 阻止组件 unmount，从而保留 state + DOM

所以在用户体验层面：
```
滚动条
表单输入
已加载数据
效果几乎一样。
```
